## Criando sua aplicação

O primeiro passo para começar a usar a Doctor Assistant AI é criar sua primeira aplicação. Acessando o 
[dashboard](https://backoffice.doctorassistant.ai), você pode criar sua primeira aplicação e realizar as configurações 
iniciais.

<Note>
Você pode ter várias aplicações, cada uma com suas próprias configurações, mas usualmente apenas uma é necessária.
</Note>

## Ambientes de execução

Ao criar sua aplicação, ela já possuirá acesso aos ambientes de execução **Sandbox** e **Production**, 
então não precisa se preocupar em criar mais aplicacoes para testar a plataforma.

- **Sandbox**: Ambiente de desenvolvimento, onde você pode testar e validar as funcionalidades da aplicação.
- **Production**: Ambiente de produção, onde a aplicação é executada em produção e é acessada por usuários reais.

No [dashboard](https://backoffice.doctorassistant.ai), para alternar entre os ambientes, você pode clicar no botão de alternância de ambiente, que está localizado 
no canto superior direito da página.

<Warning>
  Qualquer uso feito em ambiente de produção habilitará a cobrança em relação ao uso da plataforma.
</Warning>

## Configurando sua aplicação

Há alguns aspectos de configuração que são necessários para que sua aplicação funcione corretamente.
Todas essas configurações podem ser feitas no [dashboard](https://backoffice.doctorassistant.ai) na 
aba de configurações da sua aplicação, que pode ser acessada através do menu lateral esquerdo.

### Webhooks

A primeira delas é a configuração dos webhooks. Webhooks são eventos que a Doctor Assistant AI envia para a sua aplicação, 
como por exemplo, quando uma consulta é processada e concluída.

Para configurar o recebimento de webhooks, você deve fornecer uma ou mais URLs que aceitem requisições **HTTP POST**, onde 
a Doctor Assistant AI poderá enviar os eventos. É importante que a URL seja acessível publicamente e que ela retorne 
um status **200**, para que a Doctor Assistant AI possa confirmar que o evento foi recebido.

<Note>
  A Doctor Assistant AI não armazena os eventos enviados pelos webhooks, portanto, caso queira ter um histórico desses eventos, 
  você deve implementar uma lógica para isso em sua aplicação.
</Note>

### APIs

A Doctor Assistant AI possui várias APIs que podem ser utilizadas para interagir com a plataforma. O acesso a elas pode ser
ativado e desativado acessando a aba de APIs na tela de configurações da sua aplicação. Ative e desative as APIs que você
deseja utilizar de acordo com a necessidade da sua aplicação.

<CardGroup cols={1}>
  <Card title="Consultation" icon="stethoscope">
    A API de consultas permite que você envie consultas em áudio para a Doctor Assistant AI e receba sua transcrição, relatório estruturado
    e até mesmo uma lista de medicamentos recomendados para uma prescrição.
  </Card>
  <Card title="Exams" icon="file-medical">
    A API de exames permite que você envie arquivos de exames para a Doctor Assistant AI e receba um documento estruturado com todos os valores
    encontrados no exame.
  </Card>
  <Card title="Summary" icon="clipboard-list">
    A API de resumos permite que você envie um histórico de consultas e receba um resumo detalhado de todas elas.
  </Card>
</CardGroup>

<Note>
  Ao criar uma nova aplicação, todas as APIs estão desativadas por padrão. Você deve ativar as APIs que deseja utilizar.
</Note>

### Autenticação

As chaves de API são utilizadas para autenticar as requisições às APIs da Doctor Assistant AI. Elas são criadas automaticamente
quando você cria uma nova aplicação, uma chave por ambiente, e podem ser encontradas na aba de **Integração** na tela de configurações da sua aplicação.

<Warning>
  As chaves de API são sensíveis e devem ser armazenadas de forma segura em seu ambiente.
</Warning>
